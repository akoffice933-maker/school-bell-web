# 🔔 Школьный звонок Pro · Web Edition

> **Автоматический школьный звонок с голосовыми оповещениями, TTS, записью с микрофона, прямым эфиром и PWA-установкой.**
> Полностью клиентское веб-приложение на React + TypeScript + Vite + Web Audio API.
> Без сервера, без бэкенда — все данные хранятся в браузере.

## 📖 О проекте

**Школьный звонок Pro (Web Edition)** — современная веб-реализация системы автоматического школьного звонка, выполненная по техническому заданию [`SchoolBell_TZ_Tender_Sochi_2026.md`](./SchoolBell_TZ_Tender_Sochi_2026.md).

Приложение работает в любом современном браузере, не требует установки сервера или базы данных. Его можно установить как PWA на компьютер, планшет или телефон, и оно будет работать в офлайне.

### 🎯 Для кого это нужно

- 🏫 Школы, гимназии, лицеи, колледжи — для автоматизации звонков по расписанию.
- 👨‍💼 Дежурные администраторы и вахтёры — для запуска голосовых оповещений и прямого эфира.
- 🔊 Учителя и организаторы мероприятий — для воспроизведения музыки, звонков и объявлений.

### 🧠 Зачем это нужно

Система заменяет ручное управление звонками, исключает ошибки дежурного персонала, поддерживает **несколько смен**, **праздничные дни**, **повторяющиеся и однократные события** и позволяет делать **голосовые объявления** через встроенный TTS-движок браузера.

### 🌟 Ключевые преимущества

- ✅ **Zero-backend** — работает полностью в браузере, не нужен сервер.
- ✅ **PWA** — устанавливается как нативное приложение, работает офлайн.
- ✅ **Кросс-платформенность** — Windows, macOS, Linux, Android, iOS.
- ✅ **Локальное хранение** — все данные остаются на устройстве пользователя.
- ✅ **Без интернета** — после первой загрузки работает без сети.
- ✅ **Бесплатно и с открытым исходным кодом** (MIT).

## ✨ Возможности

### 📅 Расписание
- Недельное расписание с поддержкой **двух смен** (1-я и 2-я).
- **Праздничные и особые дни** — звонки отключаются или подменяются отдельным расписанием.
- **Повторяющиеся и однократные** события.
- Валидация по дате: `validFrom` / `validTo`.
- **4 готовых шаблона** расписания (1-я смена 7 уроков, 1-я смена 6 уроков, 2-я смена 7 уроков, сокращённый день 5 уроков).
- **Импорт / экспорт CSV** (для Excel) и **JSON**.
- Поиск по расписанию, фильтрация по сменам.

### 🔊 Библиотека аудио
- Загрузка **MP3 / WAV / OGG / M4A / WebM** файлов (до 50 МБ, до 10 минут).
- **3 встроенных синтетических звонка** (генерируются автоматически — на урок, с урока, на перемену).
- **Индивидуальная громкость** для каждой записи.
- Текстовые заметки / расшифровка для каждой записи.
- **Валидация** загружаемых файлов: формат, длительность, размер.
- **TTS-озвучка** — голосовые оповещения через встроенный `SpeechSynthesis` браузера (русский голос ОС).
- Поиск и фильтрация в библиотеке.

### 🎙️ Запись с микрофона
- **MediaRecorder API** — запись голоса прямо в браузере.
- **Визуализатор уровня** сигнала (RMS-метр) в реальном времени.
- Авто-сохранение записи в библиотеку.
- **Выбор микрофонного устройства** через `enumerateDevices()`.
- Возможность остановить и перезаписать.

### 📡 Прямой эфир
- Мониторинг микрофона в реальном времени через динамики.
- Можно использовать для оперативных объявлений «по школе».
- Регулировка громкости в реальном времени.

### 🛡 Служба планировщика
- Аналог `BackgroundService` + `System.Timers.Timer` из C#-версии.
- Точное срабатывание по расписанию (с защитой от перепрыгивания минут).
- Кнопка «Служба запущена / остановлена» в шапке.
- **Уведомления** в ОС при срабатывании звонка (Web Notifications API).
- **Ручной запуск** любого события из расписания.
- Автоматический пересчёт расписания каждые 5 минут (на случай долгого простоя вкладки).

### 📊 Журнал событий
- **Пагинация** (50 записей на страницу).
- Фильтрация по типу события (звонок, голосовое, прямой эфир, ручное, системное).
- Хранятся последние 500 записей.
- Полная история с временными метками.

### 🎨 Интерфейс
- 🌗 **Светлая / тёмная темы** — переключение без перезагрузки.
- 📱 **Адаптивный дизайн** — работает на мобильных, планшетах, десктопе.
- 🇷🇺 **Полная русская локализация** интерфейса.
- ♿ **Доступность** (ARIA-атрибуты, навигация с клавиатуры).
- 🎯 Боковая навигация с активным состоянием и иконками.

### 📦 PWA (Progressive Web App)
- Установка как **нативное приложение** на Windows / macOS / Linux / Android / iOS.
- **Офлайн-режим** (Service Worker кеширует все файлы).
- **Web App Manifest** с иконками 192×192 и 512×512.
- Push-стиль уведомлений (через Notification API).
- Полноэкранный режим без адресной строки браузера.

### 🔐 Безопасность
- **SHA-256** для пароля (Web Crypto API) — соответствует п. 6.3 ТЗ.
- **Никаких внешних сетевых вызовов** — всё работает локально.
- Все аудиофайлы хранятся в **IndexedDB**, метаданные — в **localStorage**.
- Полный бэкап в JSON-файл (с аудио в base64).

### 💾 Резервное копирование
- **Экспорт всего** — JSON со всеми данными + аудио в base64.
- **Импорт всего** — восстановление из JSON-файла.
- **Экспорт расписания** — только JSON-расписание (без аудио).
- **Экспорт CSV** — расписание в формате Excel.

## 🚀 Установка и запуск

### 1. Требования к системе

Перед началом убедитесь, что установлено:

| Компонент    | Минимальная версия                  | Ссылка                  |
| ------------ | ----------------------------------- | ----------------------- |
| **Node.js**  | 18.0+ (рекомендуется 20 LTS или 22) | https://nodejs.org      |
| **npm**      | 9.0+ (идёт в комплекте с Node.js)   | https://www.npmjs.com   |
| **Браузер**  | Chrome 90+ / Edge 90+ / Firefox 88+ / Safari 14+ | —     |

> 💡 **Совет:** Чтобы проверить версии, выполните `node -v` и `npm -v` в терминале.

### 2. Клонирование репозитория

```bash
# HTTPS
git clone https://github.com/akoffice933-maker/school-bell.git

# или SSH (если настроен SSH-ключ)
git clone git@github.com:akoffice933-maker/school-bell.git

cd school-bell
```

### 3. Установка зависимостей

```bash
npm install
```

Эта команда установит все пакеты, перечисленные в `package.json`:
- **react**, **react-dom** — ядро React 19.
- **clsx**, **tailwind-merge** — утилиты для условных CSS-классов.
- **@vitejs/plugin-react** — HMR и JSX-трансформация для Vite.
- **@tailwindcss/vite**, **tailwindcss** — Tailwind CSS 4 с интеграцией в Vite.
- **vite** — сборщик.
- **vite-plugin-singlefile** — сборка в один HTML-файл (опционально).
- **typescript**, **@types/*** — TypeScript и типы.

Если у вас включён **yarn** или **pnpm**, можете использовать их:

```bash
# Yarn
yarn install

# pnpm
pnpm install
```

### 4. Запуск в режиме разработки

```bash
npm run dev
```

После запуска в консоли появится что-то вроде:

```
  VITE v7.3.2  ready in 412 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.1.10:5173/
```

Откройте в браузере адрес **http://localhost:5173/**.

В режиме dev работает **горячая перезагрузка (HMR)** — изменения в коде мгновенноотображаются в браузере.

### 5. Сборка для продакшена

```bash
npm run build
```

Готовая сборка появится в папке `dist/`. Особенности:

- Бандл автоматически минифицируется (esbuild + terser).
- CSS извлекается в отдельный файл.
- `vite-plugin-singlefile` собирает всё (HTML + JS + CSS + ассеты) в **один HTML-файл**, что удобно для:
  - офлайн-использования (можно просто открыть `dist/index.html`);
  - размещения на статическом хостинге без настройки путей;
  - распространения через USB-флешку.

### 6. Просмотр продакшен-сборки локально

```bash
npm run preview
```

Поднимет мини-сервер на `http://localhost:4173/` для проверки сборки.

### 7. Деплой

Готовая папка `dist/` — это полностью статический сайт. Его можно разместить где угодно:

- **GitHub Pages** — `npm run build` → закоммитить `dist/` в ветку `gh-pages` или настроить Actions.
- **Netlify / Vercel** — подключить репозиторий, команда сборки `npm run build`, папка публикации `dist`.
- **Свой сервер** — скопировать `dist/` на nginx/apache, отдавать как статику.
- **Локально** — просто открыть `dist/index.html` в браузере (Service Worker + PWA работают в этом случае).

### 8. Установка как PWA (Progressive Web App)

После запуска `npm run build` и `npm run preview` (или деплоя на любой HTTPS-хостинг):

1. Откройте приложение в браузере.
2. В адресной строке браузера появится иконка **«Установить»** (➕ или 🖥️).
3. Нажмите на неё → **«Установить»**.
4. Приложение появится в списке программ как «Школьный звонок».

**На iOS (iPhone/iPad):**
1. Откройте в Safari.
2. Нажмите кнопку **«Поделиться»** (квадрат со стрелкой).
3. Выберите **«На экран Домой»**.

## 📖 Руководство пользователя

### Первый запуск

1. Откройте приложение в браузере.
2. Появится стартовое расписание **1-й смены** (Пн–Пт, 7 уроков) и 3 встроенных звонка.
3. **Включите службу** в правом верхнем углу — индикатор станет зелёным.
4. Звонки начнут срабатывать автоматически по расписанию.

### Разрешения браузера

При первом использовании **записи с микрофона**, **прямого эфира** или **уведомлений** браузер попросит разрешения. Их нужно дать один раз:

- 🎙️ **Микрофон** — для записи голоса и прямого эфира.
- 🔔 **Уведомления** — чтобы показывать ОС-уведомления о звонках.

### Установка расписания

- Перейдите на вкладку **«Расписание»**.
- Выберите **шаблон** (например, «Стандартная 1-я смена») — он добавит сразу все события.
- Или добавьте звонки вручную: **«+ Добавить звонок»** → укажите день, время, тип, аудио.
- Импортируйте готовое расписание из **CSV** (для Excel).

### Добавление своих звонков

- Перейдите на **«Библиотека аудио»** → **«Загрузить»** → выберите MP3/WAV/OGG.
- Или создайте **TTS-озвучку** (голосовое оповещение): нажмите «+ TTS» и введите текст.
- В расписании выберите это аудио в колонке «Аудио».

### Прямой эфир и запись

- **Запись**: вкладка «Запись с микрофона» → «Начать запись» → говорите → «Стоп» → запись автоматически попадёт в библиотеку.
- **Прямой эфир**: вкладка «Прямой эфир» → «Включить микрофон» → звук с микрофона пойдёт в колонки.

### Бэкап и перенос

- **Настройки** → **«Экспорт всего»** — сохранит `school-bell-backup.json` со ВСЕМИ данными (расписание, аудио в base64).
- **«Импорт всего»** — восстановит данные из такого файла.
- **«Экспорт расписания»** — сохранит только JSON-расписание (без аудио).

| ---------------------------------------------- | -------------------------------------------------- |
| Расписание, настройки, метаданные аудио         | `localStorage` → ключ `school_bell_state_v1`       |
| Бинарные аудиофайлы (загруженные, записанные)   | `IndexedDB` → БД `school_bell_db`, хранилище `audio_blobs` |
| Кеш TTS-голосов                                | Браузерный `speechSynthesis` (ОС)                  |
| Пароль                                         | `sha256:<hex>` в `localStorage`                    |

**Очистка данных**: Настройки → «Очистить все данные» (или вручную: DevTools → Application → Storage → Clear site data).

---

## 🔐 Безопасность

- 🔒 Пароль хешируется через **Web Crypto API (SHA-256)**, никогда не хранится в открытом виде.
- 🛡 Все загружаемые аудиофайлы проходят **строгую валидацию** (тип, размер, длительность, декодирование).
- 🚫 **Никаких внешних сетевых вызовов** — всё работает локально, в браузере.
- 💾 Данные хранятся **только на устройстве пользователя** (localStorage + IndexedDB).
- 🔄 Полный бэкап в JSON содержит **base64-аудио** — файл можно безопасно передавать.

```ts
// vite.config.ts
export default defineConfig({
  plugins: [react(), tailwindcss()], // без viteSingleFile
  // ...
});
```

## 🐛 Решение проблем

<details>
<summary><b>🔇 Звонок не воспроизводится</b></summary>

- Проверьте, что **служба запущена** (зелёный индикатор в шапке).
- Проверьте, что в **настройках** выбрана правильная смена (1-я или 2-я).
- Убедитесь, что **громкость системы** и громкость записи не на нуле.
- Откройте **журнал событий** — там видно, какие звонки сработали.
- В **консоли браузера** (F12) могут быть ошибки Web Audio API.

</details>

<details>
<summary><b>🎙️ Микрофон не записывает</b></summary>

- Дайте браузеру разрешение на использование микрофона (иконка 🔒 в адресной строке).
- Закройте другие программы, использующие микрофон (Skype, Zoom, Discord).
- В **настройках записи** ОС убедитесь, что микрофон не отключён.
- Попробуйте выбрать другое устройство в выпадающем списке.

</details>

<details>
<summary><b>🔔 Не работает TTS</b></summary>

- Убедитесь, что в системе установлен **русский голос** (Windows: Параметры → Время и язык → Язык → Русский → Параметры голоса).
- На Linux: установите `espeak` или `festival` с русским голосом.
- В **Chrome/Edge**: `chrome://settings/languages` → добавьте русский.

</details>

<details>
<summary><b>💾 Не сохраняются данные</b></summary>

- Браузер может очищать `localStorage` в режиме инкогнито. Используйте обычное окно.
- Проверьте, что в браузере не включена опция «Очищать cookies при закрытии».
- Не используйте **Brave** со строгой блокировкой IndexedDB.

</details>

<details>
<summary><b>📡 PWA не устанавливается</b></summary>

- Установка работает только по **HTTPS** или на `localhost`.
- В **iOS Safari** нет кнопки «Установить» — нужно вручную: «Поделиться» → «На экран Домой».
- В некоторых браузерах PWA отключена в `about:config` / `chrome://flags`.

</details>
