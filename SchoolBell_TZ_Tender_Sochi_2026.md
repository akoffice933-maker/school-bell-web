## 1. Общие сведения о проекте

### 1.1 Наименование системы

- **Полное наименование:** «Автоматический школьный звонок с голосовыми оповещениями»
- **Краткое наименование:** SchoolBell Pro

### 1.2 Заказчик

| Параметр | Значение |
|---|---|
| Тип организации | Образовательное учреждение / администрация школы |
| Регион | г. Сочи, Краснодарский край, РФ |
| Место развёртывания | Локальная сеть образовательного учреждения |
| Целевая ОС | Windows 10 / Windows 11 Pro (x64) |

### 1.3 Назначение системы

Система предназначена для автоматизации управления школьными звонками и звуковыми оповещениями в образовательных учреждениях. ПО обеспечивает:

- Воспроизведение аудиосигналов по расписанию (начало/конец урока, перемены, события)
- Управление пользовательской библиотекой аудиофайлов
- Запись и воспроизведение голосовых оповещений с микрофона
- Централизованное управление через графический интерфейс (WPF)
- Фоновую работу через службу Windows (Windows Service) с автозапуском
- Ведение журнала событий и логирование

## 2. Цели и задачи разработки

### 2.1 Цель разработки

Создание готового к использованию, надёжного Windows-приложения для автоматизации системы школьных звонков с поддержкой пользовательских аудиофайлов, записи голоса и гибкого управления расписанием.

### 2.2 Задачи

1. Реализовать планировщик звонков с поддержкой многосменного расписания, праздников и разовых событий.
2. Создать библиотеку аудиофайлов с загрузкой пользовательских MP3/WAV и управлением из GUI.
3. Интегрировать запись с микрофона (NAudio), сохранение голосовых аннотаций и прямой эфир.
4. Разработать Windows Service (BackgroundService / .NET 8) с автозапуском и уведомлениями в трее.
5. Построить WPF-интерфейс с темами (светлая/тёмная), просмотром логов и экспортом/импортом расписания.
6. Обеспечить хранение данных в SQLite через Entity Framework Core 8 с поддержкой миграций.
7. Подготовить инсталлятор (Inno Setup) и документацию пользователя/администратора.

### 2.3 Ожидаемый результат

Полностью работоспособное приложение, готовое к установке и эксплуатации на Windows 10/11 без дополнительных зависимостей (self-contained publish). Код — чистый, структурированный, с комментариями, покрытый базовыми Unit-тестами.

├── SchoolBell.Core          # Class Library — модели, интерфейсы, репозитории, бизнес-сервисы

## 5. Модели данных (Entity Framework Core)

### 5.1 Сущности базы данных

| Сущность | Ключевые поля | Назначение |
|---|---|---|
| `AudioFile` | `Id`, `FileName`, `FilePath`, `DurationSeconds`, `Volume`, `SourceType` (Uploaded/RecordedFromMic), `TranscriptText`, `IsInUse` | Библиотека аудиофайлов |
| `BellType` | `Id`, `Name`, `Category` (Bell/VoiceAnnouncement) | Типы событий звонка |
| `Schedule` | `Id`, `DayOfWeek`, `Time`, `BellTypeId`, `AudioFileId`, `Shift`, `IsRecurring`, `ValidFrom`, `ValidTo` | Расписание звонков |
| `Holiday` | `Id`, `Date`, `IsBellDisabled`, `CustomScheduleJson` | Праздники и особые дни |
| `MicRecording` | `Id`, `AudioFileId`, `RecordingDate`, `DurationSeconds`, `TranscriptText`, `WasLiveBroadcast` | Журнал записей с микрофона |
| `LogEntry` | `Id`, `Timestamp`, `ScheduleId`, `AudioFileName`, `Message` | Журнал срабатываний |
| `Settings` | `Id`, `ServiceEnabled`, `AudioDeviceId`, `DefaultVolume`, `Theme`, `PasswordHash` | Глобальные настройки |

### 5.2 Строка подключения

База данных по умолчанию:

```
%ProgramData%\SchoolBell\bell.db
```

Строка подключения в `appsettings.json`:

```json
{
  "ConnectionStrings": {
    "Default": "Data Source=%ProgramData%/SchoolBell/bell.db"
  }
}
```

Директория создаётся автоматически при первом запуске. Миграции EF Core применяются при старте службы.

## 6. Нефункциональные требования

### 6.1 Надёжность

- Служба автоматически перезапускается при сбое (`RecoveryOptions`: restart after 60 с).
- При недоступности аудиофайла — запись в лог и пропуск события без аварийного завершения.
- Максимальное время пропуска звонка относительно расписания: **не более 5 секунд**.

### 6.2 Производительность

- Задержка воспроизведения аудио после наступления времени события: **≤ 500 мс**.
- Задержка прямого эфира (микрофон → динамик): **≤ 200 мс**.
- Потребление CPU в фоновом режиме (idle): **не более 3%**.

### 6.3 Безопасность

- Хранение пароля защиты настроек: SHA-256 хэш (не plaintext).
- Аудиофайлы хранятся локально, без сетевого доступа к сторонним сервисам.
- Логи не содержат персональных данных.

### 6.4 Сопровождаемость

- Код покрыт комментариями на русском или английском языке.
- SOLID-принципы, Dependency Injection, Repository Pattern.
- Unit-тесты: покрытие **не менее 40%** для `SchoolBell.Core` и `SchoolBell.Audio`.

### 6.5 Установка и развёртывание

- Инсталлятор Inno Setup: одним файлом (`.exe`), автоустановка службы, создание ярлыков.
- Self-contained publish — работа без предустановленного .NET Runtime.
- Поддержка обновления (upgrade) без потери БД и аудиофайлов.
